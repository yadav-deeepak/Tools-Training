‚ùÑÔ∏è Concept of branches in github
##################################
‚û§ Whenever we want to add some new feature to our application or we want to fix some bug we dont work directly on the main branch code 
‚û§ Main branch code is used for the deployement it is used to run our project so we dont work directly on the main branch because suppose if we start to work directly on main branch and we are developing a feature so if we push the incomplete feature to our main branch it will affect our production server and our code might stop running 
‚û§ So what we do is we create a clone of project then we create the branch from our main and then we start working on that branch we will push our daily work there this way it wont affect our main branch and when our feature development is complete the tech lead will merge our branch to the main branch.
‚û§ So like this many branches are created from the main branch and everyone can work parallely on differnt task .
‚û§ Every on is working on there own branch when they are finished with the task then they will tell the tech lead and he/she will merge the branch to the main branch this way it wont affect the production code and new features are also added .
